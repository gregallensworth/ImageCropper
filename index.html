Check out index.js and you'll see the following:
    $('input[type="file"][name="image"]').ImageCropper({
        width: 800,
        height: 600,
        thumbWidth: 200,
        thumbHeight: 150,
        onImageLoaded: function (rawimg, $cropcanvas, $fileinput) {
            $('#bullets').hide();
            $('#instructions').show();
            $('#done').hide();
        },
        onImageCropped: function (rawimg, $hiddenfield, $fileinput) {
            $('#bullets').hide();
            $('#instructions').hide();
            $('#done').show();
        }
    });
  • The image[type="file"] element gains a new behavior: selecting a file opens a image-cropping utility.
    Pro tip: Consider using a accept= attribute on your file input, e.g. <input type="file" name="image" accept=".jpg,.jpeg,.png" />
  • The size of this image-cropper panel (800x600 by default) determines the size of the cropped image, so the user is seeing an accurate representation of the final result.
  • The CSS is intentionally minimal, so as not to conflict with your page content and styling. See ImageCropper.css which comes with the bundle, for a list of all CSS classes you may find interesting.
  • When the user confirms with the "Crop & Save" button, the image[type="file"] is emptied and hidden, and a input[type="hidden"] field is created in its place, containing the base64-encoded data. This is optional.
  • You may supply optional callback functions when an image is loaded, and when an image is cropped. Here, they are used to show instructions on the screen.
